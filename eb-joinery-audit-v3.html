New Findings — v3

Issues introduced or discovered in the new files

Critical
Landing pages missing mobile CTA bar — 4 pages regress mobile UX

joinery-services-widnes.html, home-improvements-liverpool.html, bespoke-joinery-cheshire.html, and areas-we-cover.html all have zero instances of mobile-cta. These are likely your highest-traffic SEO pages and the most probable first landing point for mobile users searching by location — yet they're the only pages without the sticky Call Now bar.

Fix: Copy the .mobile-cta-bar block from any main page into the <body> of all four files, and add padding-bottom: 5rem to the footer on each.

Critical
Landing pages have no BreadcrumbList schema

All four landing pages and areas-we-cover.html carry LocalBusiness and Service schema but no BreadcrumbList. These pages benefit most from breadcrumb rich results in Google — they're the targeted local search pages. services.html does have BreadcrumbList correctly. The gap is specifically on the four geo-targeted files.

Fix: Add a BreadcrumbList JSON-LD block to each: Home → [Page Name], pointing to the correct canonical URL.

Critical
Landing pages missing geo meta tags entirely

The geo.region / geo.placename / geo.position meta tags added in v2 are present on services.html but absent from all four new landing pages. This is particularly notable for joinery-services-widnes.html and bespoke-joinery-cheshire.html — pages whose entire purpose is local geographic targeting. They have no geo signal at all in the meta layer.

Fix: Add the three geo meta tags to each page's <head>, adjusting placename and position per page (e.g. Liverpool: 53.4084;-2.9916, Cheshire: 53.1903;-2.8923).

Moderate
Blog internal links still use query strings despite clean slug redirects existing

The _redirects file correctly maps clean slugs like /blog/5-ways-to-add-value-to-your-home to /blog-post.html?id=1. However, main.js still generates all blog links with blog-post.html?id=${p.id}. This means Google crawls the query string URL, not the clean slug. The redirect is in the wrong direction for SEO — you want the clean slug to be the canonical, not a redirect target.

Fix: In main.js, add a slug map object and generate links as /blog/[slug]. Update the blog post canonical in setMeta() at line 520 to use the clean slug too. Reverse the _redirects so the clean slug is the destination, not the source — or keep as 301 but ensure the canonical points to the clean URL.

Moderate
Landing pages missing scroll cue — 100svh hero with no scroll signal

The four landing pages all use a full-viewport hero with no .scroll-cue element. The scroll cue was implemented on the main pages in v2 precisely because users couldn't tell there was content below the fold. The same problem exists here — and arguably matters more, since these pages need users to scroll down to the service cards and CTA sections that drive conversions.

Fix: Add the .scroll-cue SVG chevron element (already in style.css) to the hero section of each landing page.

Moderate
Sitemap omits clean blog slug URLs — crawlers only see blog.html

The sitemap correctly includes blog.html but contains no entries for individual blog posts. Now that clean slugs exist via _redirects, the four blog post URLs (/blog/5-ways-to-add-value-to-your-home etc.) should be in the sitemap. Without this, Google has no direct crawl path to the individual posts — it has to discover them by crawling the blog listing page and following JS-rendered links.

Fix: Add the four clean blog slug URLs to sitemap.xml with changefreq: monthly, priority: 0.6.

Minor
areas-we-cover.html has no internal links to the three geo landing pages

The Areas We Cover page mentions Widnes, Liverpool, Warrington, and Cheshire in its content sections but doesn't link to the dedicated landing pages for those regions. This is a missed internal linking opportunity — a user reading about Liverpool coverage would naturally click through to the Liverpool-specific page, and Google would see the topical connection between the pages.

Fix: Add contextual links from each area card to its corresponding landing page: Widnes → joinery-services-widnes.html, Liverpool → home-improvements-liverpool.html, Cheshire → bespoke-joinery-cheshire.html.

Minor
Landing pages expose /admin.html in navbar — publicly visible link

All four landing pages include <a href="/admin.html"> in the navigation. While admin.html is noindex and login-protected, surfacing the admin link publicly on every page is unnecessary and unprofessional. A determined user or competitor could find and repeatedly probe the admin login. The main pages likely have this too, but it's worth removing from all pages.

Fix: Remove the admin link from all nav elements. Access admin directly by URL when needed.

Persistent Issues — Carried from v2

Still outstanding from the last round

Critical
JS-only portfolio — #main-portfolio-grid empty in HTML source

Portfolio content is still entirely JavaScript-rendered. The HTML source contains an empty #main-portfolio-grid div. Google's crawler may not execute JS or may delay rendering, making portfolio content invisible to the index. This is the single highest-impact remaining SEO issue on the site.

Fix: Pre-render 12 static portfolio cards as crawlable HTML baseline in portfolio.html. JS can still hydrate and extend dynamically on top.

Moderate
Accent colour incomplete — .section-title em and .about-quote still use var(--accent)

.service-link (line 676) was updated to var(--accent-on-light) correctly. But .section-title em (line 555) and .about-quote (line 874) still reference var(--accent) — which is #38bdf8, a light blue with only ~2.8:1 contrast on white. These appear on white-background sections and fail WCAG AA.

Fix: Change both to var(--accent-on-light) (#1a5f7a — 5.9:1 contrast). Two-line fix in style.css.

Moderate
CLS risk — .portfolio-card has no aspect-ratio container

.portfolio-card img has aspect-ratio: 16/9 at mobile breakpoints (line 2093) but the card container itself has no explicit aspect-ratio or fixed height. Without a known container size, the browser can't reserve space before the image loads, causing layout shift scored against Core Web Vitals.

Fix: Add aspect-ratio: 4/3 to .portfolio-card, .portfolio-item at lines 736–746.

Moderate
Toast notification obscured by mobile CTA bar

The #toast element is fixed at bottom: 2rem. The mobile CTA bar is fixed at the bottom with a height of approximately 3.5–4rem. On mobile, a successful form submission toast will appear beneath the CTA bar — invisible to the user who just submitted the form.

Fix: Add @media (max-width: 900px) { .toast { bottom: 5rem; } } to style.css.

Summary

Where things stand after three rounds

The most significant SEO wins this round are the clean blog slug redirects and the complete sitemap — both solid additions. The landing pages are well-structured with good schema and OG tags, and the design system is applied consistently across all of them.

The drop in mobile score is a direct consequence of deploying four new high-traffic pages without the mobile CTA bar and scroll cue that exist on every main page. This needs to be the first thing fixed — these are your primary local search landing pages.

After resolving the mobile CTA gap on the landing pages and adding geo meta tags and BreadcrumbList to match, the site moves decisively into upper-mid territory across all three categories. The remaining persistent issues (portfolio static render, CLS, toast) are the last technical quality flags before this would pass a rigorous agency QA checklist.
